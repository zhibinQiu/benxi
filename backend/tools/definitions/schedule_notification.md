---
name: schedule_notification
---
安排一条定时站内通知，在指定时间送达用户。
当用户说「X秒/分钟后提醒我」「定时通知我」「设置提醒」时使用。

## When to use
- 用户说「X秒/分钟后提醒我」
- 用户说「定时通知我」「设置提醒」「到时间告诉我」
- 需要安排未来某个时间点发送通知

## When NOT to use
- 需要立即发送（用 send_notification，不要用 schedule_notification 传当前时间）
- 发送外部邮件、短信、第三方应用消息（不支持）
- 周期重复提醒如「每天提醒」（不支持，需多次调用）
- 创建待办任务（用 create_todo）

## Returns
- 定时通知 ID 和计划发送时间

## Parameters

### title (required)
通知标题，简短概括提醒内容。最长 200 字符。

### body (optional)
通知正文，补充说明提醒详情。最长 4000 字符。不需要时留空。

### link (optional)
点击通知后跳转的链接。不需要时留空。

### scheduled_at (required)
ISO 8601 绝对时间（含时区），如 `2026-07-09T09:30:00+08:00`。
用户说相对时间（如「30秒后」）时，你必须自行计算出绝对时间再传此字段。
不要传相对时间表达式，只传计算后的 ISO 8601 绝对时间。
时区偏移不可省略（东八区用 +08:00，UTC 用 Z）。
