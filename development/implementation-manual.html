    5. 核心设计原则（实现时必须遵守）

    1. 平台 ACL 是文档权限的唯一真相；KnowFlow/RAGFlow 只做解析、检索与 UI，检索前必须先算 allowed_document_ids（见 知识服务实现）。
    2. API 薄、服务厚；新 HTTP 接口不写复杂 ORM 编排，放 services/ 或 domains/（见 分层架构）。
    3. 知识能力走 KnowledgeGateway；禁止在新代码中散落 from app.services.ragflow_sync_service import ...（见 应用服务与域）。
    4. 功能入口走插件；不在 MainLayout 硬编码业务菜单（见 功能插件）。
    5. 统一响应与中文错误；ApiResponse + AppError + 前端 parseResponse / sanitizeUserFacingMessage（见 API 与约定）。
    6. 大文件与长任务异步化；上传走 MinIO presigned；翻译/对比/删文档走 Celery 或 BackgroundTasks（见 异步任务）。

    6. 配置与环境变量索引

    完整列表见 platform/.env.example 与 配置说明。高频项：

    变量 用途
    DATABASE_URL / REDIS_URL / MINIO_* 基础设施
    JWT_SECRET 生产必换
    PDF2ZH_API_URL 翻译服务地址
    KNOWFLOW_ENABLED 是否集成知识栈
    RAGFLOW_API_URL / KNOWFLOW_BACKEND_URL RAGFlow UI/API 与 KnowFlow 扩展 API
    RAGFLOW_ACCOUNT_MODE mapped（推荐）/ shared
    RAGFLOW_SYNC_ON_LOGIN / RAGFLOW_SYNC_ON_EMBED 同步时机
    DEEPSEEK_* 摘要、会议总结
    SPEECH_SERVICE_URL 语音转写

    变更类型 更新文档
