# 可行性研究报告（可研）

## 适用场景

- 项目可行性研究、投资可行性分析、建设必要性论证
- 用户明确提到「可研」「可行性研究」「可行性分析报告」

## 工作流程

1. **确认范围**：项目名称、建设地点、投资主体、编制依据（若用户未给，在文中标注「待补充」）。
2. **收集材料**（按需）：
   - 企业/政策材料 → `knowledge_retrieve`
   - 市场数据、同类项目、最新政策 → `web_search`（含「最新动态」「近期政策」类查询）
3. **加载模板**：阅读本 Skill 的 `templates/outline.md`，按章节顺序撰写。
4. **输出长文**：Markdown 标题结构；正文不少于 **5000 字**；句末用 `[n]` 标注引用，编号与检索材料严格对应。
5. **质量自检**：各章有实质内容；不编造文号与数据；无 AI 套话（「综上所述」「赋能」等）。

## 章节结构

见 `templates/outline.md`：摘要 → 项目概况 → 建设必要性 → 市场分析 → 技术方案 → 投资估算 → 财务评价 → 风险分析 → 结论与建议。

## 与其他报告 Skill 的区分

| 用户表述 | 应选 Skill |
|----------|------------|
| 需求分析、PRD | `report-requirements` |
| 建设/实施方案 | `report-construction-plan` |
| 调研、行业研究 | `report-survey` |
| 测试、验收 | `report-test` |
| 工作计划 | `report-work-plan` |

若类型不明确，先向用户确认 1 句，或按可研默认结构撰写并在摘要说明假设。
