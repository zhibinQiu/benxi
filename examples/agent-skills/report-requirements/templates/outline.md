# 需求分析报告 — 章节大纲

## 摘要

背景、分析目标、核心需求结论、建议优先级。

## 一、项目背景与目标

- 业务背景与动因
- 项目目标与成功标准
- 范围与边界（含不在范围内事项）

## 二、干系人与用户分析

- 角色画像与使用场景
- 利益相关方诉求

## 三、现状与问题分析

- 现有流程/系统现状
- 痛点与差距（可结合数据或案例）

## 四、业务需求

- 业务流程描述（可用文字或简要步骤）
- 业务规则与约束

## 五、功能需求

- 功能清单（编号 + 描述 + 优先级）
- 关键用例或用户故事（按需）

## 六、非功能需求

- 性能、安全、可用性、合规、集成等
- 验收口径

## 七、数据与接口需求

- 主要数据实体与来源
- 外部系统/接口依赖（材料不足标「待补充」）

## 八、风险、假设与依赖

- 关键假设
- 风险与缓解措施

## 九、结论与建议

- 需求优先级汇总
- 分期实施建议
