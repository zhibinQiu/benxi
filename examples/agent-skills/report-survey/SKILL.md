# 调研 / 研究报告

## 适用场景

- 行业调研、市场研究、专题研究报告、政策研究、竞品/现状调研
- 用户说「研究报告」「调研报告」「行业分析」且未明确其他类型时的**默认** Skill

## 工作流程

1. **判断类型**：若用户明确可研/需求/建设/测试/工作计划，应改用对应 Skill，勿混用结构。
2. **多角度检索**：
   - `knowledge_retrieve`：本地制度、历史材料
   - `web_search`：至少包含主题、最新动态、近期新闻/政策类查询
3. **按 `templates/outline.md` 撰写**，正文 ≥5000 字，融合政策、市场、技术、案例、风险等多视角。
4. **图文并茂**：至少 3 张 Markdown 表格 + 1～2 个 Mermaid 图（产业链 mindmap、竞争格局 flowchart 等）；数据与对比必须用表，勿纯文字重复。
5. **充实度**：每节须多段论述，禁止标题下仅一两句话；禁止车轱辘话与空泛套话。
6. **引用**：句末 `[n]`；正文不写文档名/URL；冲突时本地材料优先于联网。
